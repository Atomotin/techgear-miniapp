# Как теперь управлять товарами

Теперь основной способ управления каталогом:

1. Запусти сервер: `npm start`
2. Открой админ-панель: `http://localhost:3000/admin`
3. Введи пароль из переменной `ADMIN_PASSWORD`
4. Добавляй, редактируй и удаляй товары через интерфейс

## Первый запуск

При первом запуске сервер сам создаст:

- `data/catalog.json` - каталог товаров и категорий
- `data/orders.json` - заказы из Mini App

Каталог автоматически переносится из текущего файла [card-tovary.js](c:\Users\User\Desktop\techgear-miniapp\card-tovary.js), чтобы не потерять существующие товары.

## Что можно делать в админке

- добавлять новый товар
- редактировать существующий товар
- скрывать товар с витрины
- помечать товар как `new` или `hot`
- менять порядок сортировки
- редактировать список изображений
- добавлять и удалять категории
- смотреть заказы
- менять статус заказа: `new`, `processing`, `done`, `cancelled`

## Как задавать изображения

Сейчас в админ-панели изображения указываются путями, по одному на строку.

Пример:

```text
images.img/photo_1.jpg
images.img/photo_2.jpg
images.img/photo_3.jpg
```

Первая строка станет основной картинкой товара.

## Пароль админки

Создай свои переменные окружения на основе файла [.env.example](c:\Users\User\Desktop\techgear-miniapp\.env.example):

```text
ADMIN_PASSWORD=your-strong-password
ADMIN_SECRET=another-long-random-secret
PORT=3000
```

Если переменные не заданы, сервер использует пароль по умолчанию `techgear-admin`. Для реального использования его нужно обязательно поменять.

## Важно

- [card-tovary.js](c:\Users\User\Desktop\techgear-miniapp\card-tovary.js) теперь нужен только как стартовый источник данных для миграции
- после запуска и создания `data/catalog.json` витрина будет брать каталог из API сервера
- если сервер недоступен, витрина откатится к локальному каталогу как fallback
